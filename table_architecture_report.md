# Отчёт: Архитектура и настройки таблиц (v1.9.5)

Этот отчёт описывает разделение ответственности между универсальным компонентом `VirtualSmartTable` и конкретными страницами (`MastersPage`, `WarehousePage`).

---

## 1. Общие настройки (VirtualSmartTable.tsx)
Эти функции и параметры работают **одинаково** для всех таблиц в приложении:

*   **Виртуализация**: Использование `@tanstack/react-virtual` для плавной отрисовки тысяч строк без лагов.
*   **Сетка (Grid Layout)**: Автоматический расчет ширины колонок на основе параметров `w-[Xpx]` в определении колонок.
*   **Липкие колонки (Sticky)**: Система "замораживания" колонок (например, Индекс или Имя) слева или справа с тенями (`FREEZE_SHADOW`).
*   **Режим выделения (Selection Mode)**: 
    *   Логика чекбоксов и массового выделения. 
    *   **Унифицированный клик**: Если выделена хотя бы одна строка, клик по любому месту другой строки переключает её выбор.
*   **Перенос текста (Word Wrap)**: Глобальный переключатель между однострочным режимом (`truncate`) и многострочным (`whitespace-pre-wrap`).
*   **Сортировка и поиск**: Интерфейс заголовков, иконки направления сортировки и подсветка активной колонки поиска.
*   **Drag-and-drop**: Возможность менять колонки местами перетаскиванием.

---

## 2. Специфика Склада (WarehousePage.tsx)
Логика, которая встроена в таблицу, но активируется только для данных склада или по флагу `isProductView`:

*   **Иконки папок**: Отображение `FolderTre`, `FolderCog` и счётчиков вложенности.
*   **Индикаторы состояния**: Специфические иконки 📉 (отрицательный остаток), 💸 (проблемы с ценой), 📦 (нулевой остаток).
*   **Цветовая индикация**: Красный цвет для отрицательных чисел в колонке `stock`.
*   **Расчеты на лету**: Встроенный расчет `Profit`, `Margin` и `ROI`, если страница не передала свои значения.

---

## 3. Специфика Мастеров (MastersPage.tsx)
Вся уникальная логика Мастеров вынесена **из компонента таблицы** и передается через функцию `getCellValue`. Это позволяет менять логику мастеров, не ломая склад:

*   **Чистка имен**: Вырезание системных символов из имен (управлятся локальным состоянием `showRawNames`).
*   **Стаж (Seniority)**: Отрисовка цветных кругов с буквами (М, Г, В) и **тултипами**, содержащими точную дату.
*   **Типы контактов**: Логика определения системного контакта (`Building2`) или архивного мастера (`UserMinus`).
*   **Бейджи ("1")**: Уникальный индикатор для архивных мастеров в стиле складских папок.
*   **Категории**: Объединение полей `category` и `parsedCategory`.
*   **Подсветка телефонов**: Жирный красный шрифт для номеров, помеченных `!!!`.

---

## 4. Механизм взаимодействия
Связующим звеном является паттерн **"Fallback Rendering"**:
1. Таблица спрашивает: *"Есть ли у страницы кастомное значение для этой ячейки (getCellValue)?"*
2. Если страница (например, Мастера) возвращает JSX — рисуется он.
3. Если страница вернула `null` или функции нет — таблица включает свою "умную" встроенную логику (складскую).
4. Если и там ничего нет — рисуется прочерк `—`.

> [!NOTE]
> Такая архитектура позволяет добавлять новые страницы (например, "Клиенты" или "Заказы") просто создав новый `getCellValue`, не меняя код самого компонента `VirtualSmartTable`.
